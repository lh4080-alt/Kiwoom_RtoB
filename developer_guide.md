# 키움 REST API 개발 가이드라인 (Developer Guidelines)

이 문서는 키움증권 REST API를 사용하여 프로그램을 개발하거나 유지보수할 때 반드시 준수해야 할 사항들과, 공식 문서를 꼼꼼히 보지 않으면 놓치기 쉬운 데이터 제약 사항들을 정리한 문서입니다.

## 1. 필수 구현 로직 (Critical Implementation Logic)

시스템 안정성과 데이터 무결성을 위해 아래 로직은 반드시 코드에 포함되어야 합니다.

### 1.1 조건검색 실시간 등록 시퀀스 준수
* **관련 API:** `ka10173` (조건검색 요청 실시간), `ka10171` (조건검색 목록조회)
* [cite_start]**필수 규칙:** 실시간 조건검색 등록(`CNSRREQ`)을 요청하기 전에는, **반드시 조건검색 목록조회(`CNSRLST`) API를 먼저 호출**하여 응답을 받아야 합니다. [cite: 318, 321]
* **이유:** 키움증권 시스템 내부 구조상, 목록 조회가 선행되지 않은 상태에서 실시간 등록을 요청하면 데이터 수신이 불가능하거나 에러가 발생합니다.
* **구현 위치:** `automation/realtime/websocket.py`의 `_ensure_condition_list_loaded` 메서드 참고.

### 1.2 설정 파일 경로 규칙
* **규칙:** 모든 사용자 설정 파일(`settings.json`, `sold_stocks.json`, `trailing_status.json` 등)은 반드시 **`config/data/` 디렉토리** 내에 저장하고 관리해야 합니다.
* **이유:** 소스 코드 업데이트 시 사용자 설정이 초기화되는 것을 방지하고, 설정 파일의 관리를 일원화하기 위함입니다.
* **참고:** 프로그램 시작 시 구 버전 경로(`config/`)의 파일을 `config/data/`로 자동 마이그레이션하는 로직이 `main.py`에 포함되어 있습니다.

## 3. 리소스 및 성능 관리 (Resource & Performance)

### 3.1 요청 제한 속도 (Rate Limiting)
* **규칙:** REST API 호출 시 **초당 5회 미만** (분당 약 300회)의 속도 제한을 준수해야 합니다.
* **구현:** `automation/utils/rate_limiter.py`에 구현된 `AsyncRateLimiter`를 사용하여 모든 HTTP 요청을 제어해야 합니다. 이를 어길 시 API 호출이 차단될 수 있습니다.

### 3.2 웹소켓 연결 유지 (Keep-Alive)
* **규칙:** 서버로부터 `PING` 메시지를 수신하면, 즉시 `PONG` (또는 수신한 메시지 그대로)을 전송하여 연결이 끊어지지 않도록 해야 합니다.
* **재접속:** 장 운영 시간 중 네트워크 불안정으로 연결이 끊길 경우를 대비한 자동 재접속 로직이 필수입니다.

## 4. 인증 및 운영 (Authentication & Operation)

* **토큰(Token) 수명:** Access Token의 유효 기간은 통상적으로 **발급 후 폐기 전까지(최대 1일)**입니다.
* **재로그인:** 안정적인 자동매매를 위해 **매일 장 시작 전(예: 08:30)**에 프로그램을 재시작하거나, 내부 스케줄러를 통해 토큰을 갱신하고 웹소켓을 재연결하는 과정이 필요합니다.
* **모의/실전 구분:** `config/` 내의 App Key와 Secret Key는 모의투자와 실전투자가 서로 다르므로, 실행 모드(`is_paper_trading`)에 맞춰 올바른 키를 로드해야 합니다.
